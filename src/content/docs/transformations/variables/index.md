**Variables** let you parametrize a transformation so one configuration can serve several cases; **shared code** lets you reuse a snippet across transformations. Both are evaluated before the transformation runs, so they don't interfere with your code at runtime.

This page is split by what you need:

- **[How do I use variables and shared code?](/transformations/variables/how-to/)** — define variables and defaults, override them at runtime or per flow, and create, reuse, and manage shared code.
- **[What are variables and shared code?](/transformations/variables/explanation/)** — the concepts and the inline-vs-linked trade-off.
