Kai is Keboola's embedded AI assistant— a data engineering co-pilot that understands your project inside and out. Unlike general AI tools, Kai has direct access to your configurations, schemas, job logs, and data lineage, providing specific, actionable answers instead of generic advice.

**Kai isn't a generic chatbot. It's connected to your project.**

* Reads your configs, schemas, and logs automatically
* Suggests fixes based on your actual setup
* Takes action when you approve

## Who Should Use Kai

* Data engineers building and maintaining data pipelines
* Analytics engineers creating SQL and Python transformations
* Project managers monitoring jobs and debugging issues
* New team members learning Keboola best practices
* Anyone working with data who wants to move faster

## What You Can Do With Kai

### Analyze & Explore
Execute SQL queries, explore schemas, calculate metrics, analyze trends.

*"Show me total revenue by month from the transactions table"*

### Build & Configure
Set up extractors, writers, transformations, and custom components.

*"Create a Snowflake writer for the customer_summary table"*

### Debug & Fix
Diagnose failures, resolve errors, fix bugs, investigate data issues.

*"Why did flow 'daily-etl' fail last night?"*

### Document & Model
Generate documentation, describe tables, design data models.

*"Document what transformation 'revenue-calc' does"*

## How Kai Differs from Other AI Tools

| | ChatGPT/Claude Web | Claude + MCP Server | Kai |
|---|---|---|---|
| **Setup** | None needed | Install MCP server locally | None needed |
| **Access to Keboola** | ❌ Copy-paste only | ✅ Direct via MCP | ✅ Built-in |
| **Reads configurations** | ❌ You must paste them | ✅ Automatic | ✅ Automatic |
| **Reads data schemas** | ❌ You must describe them | ✅ Automatic | ✅ Automatic |
| **Takes actions** | ❌ Manual copy-paste | ✅ With approval | ✅ With approval |
| **Context awareness** | ❌ You provide context | ⚠️ Via MCP only | ✅ Knows where you are |
| **Audit trail** | ❌ None | ⚠️ Separate logs | ✅ Activity Center |
| **Works in browser** | ✅ Yes | ❌ Desktop app only | ✅ Yes |
| **Dev branch support** | ❌ No | ❌ No | ✅ Yes |

*In short: Kai brings MCP-like capabilities directly into your browser with deeper platform integration and no setup required.*

## Getting Started

* **Pay-as-you-go clients:** Kai is now in Public Beta and available to all pay-as-you-go users. Look for the Kai button in your project's navigation bar.
* **Multi-tenant clients:** Organization Admins can enable Kai from the chat screen or via Settings → Features
* **Single-tenant users:** Coordinate with your Customer Success Manager or [contact Keboola Support](mailto:support@keboola.com)
